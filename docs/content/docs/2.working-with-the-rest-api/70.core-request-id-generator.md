`RequestIdGenerator`{lang="ts-type"} is the default `IRequestIdGenerator`{lang="ts-type"} implementation used by the HTTP clients to:

- generate a unique id per outgoing request via [`Text.getUuidRfc4122()`](/docs/working-with-the-rest-api/tools-text/) (UUID v7),
- decide which HTTP header carries it (`X-Request-ID`),
- decide which query string parameters carry it and the SDK metadata (`bx24_request_id`, `bx24_sdk_ver`, `bx24_sdk_type`).

You only override this when you want server-side correlation with an existing tracing scheme (e.g. propagate a request id from an upstream service) — implement `IRequestIdGenerator`{lang="ts-type"} and inject it through a custom `Http` setup.

## Default Constants

| Constant | Value |
|----|----|
| Header field | `X-Request-ID` |
| Query parameter | `bx24_request_id` |
| Query parameter (SDK version) | `bx24_sdk_ver` |
| Query parameter (SDK type) | `bx24_sdk_type` |

The latter two are constant per build (replaced at compile time by `unbuild`).

## API

```ts-type
class RequestIdGenerator implements IRequestIdGenerator {
  getRequestId(): string                       // UUID v7
  getHeaderFieldName(): string                 // 'X-Request-ID'
  getQueryStringParameterName(): string        // 'bx24_request_id'
  getQueryStringSdkParameterName(): string     // 'bx24_sdk_ver'
  getQueryStringSdkTypeParameterName(): string // 'bx24_sdk_type'
}
```

## `IRequestIdGenerator` Interface

```ts-type
interface IRequestIdGenerator {
  getRequestId(): string
  getHeaderFieldName(): string
  getQueryStringParameterName(): string
  getQueryStringSdkParameterName(): string
}
```

If you supply your own implementation, ensure `getRequestId()` returns a value that is **unique per call** — the limiter and the request log key on it.
