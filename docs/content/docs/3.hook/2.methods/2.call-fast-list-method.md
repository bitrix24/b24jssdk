## Quick Overview

Use `callFastListMethod`{lang="ts-type"} for quick data retrieval. Not count the total number.
The method returns a `Promise` with an `Result`{lang="ts-type"} object containing the response data.

```ts
// Basic usage
import { B24Hook } from '@bitrix24/b24jssdk'

const $b24 = B24Hook.fromWebhookUrl('https://your_domain.bitrix24.com/rest/1/webhook_code/')

const response = await $b24.callFastListMethod('method.api', {parameters}, 'id', 'items')
```

## Method Signature

```ts-type
callFastListMethod<T = unknown>(
  method: string,
  params: {
    order?: any
    filter?: any
    [key: string]: any
  },
  idKey: string,
  customKeyForResult?: string | null
): Promise<Result<T[]>>
```

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| **`method`** | `string`{lang="ts-type"} | Yes | Name of the REST API method to call (e.g. `crm.item.get`). |
| **`params`** | `object`{lang="ts-type"} | No | Object with parameters to pass to the API method. |
| **`idKey`** | `string`{lang="ts-type"} | No | Entity `ID` field name (e.g. `ID`, `id`). Default `ID`. |
| **`customKeyForResult`** | `string`{lang="ts-type"} | No | Custom field indicating that the result will be a grouping key (e.g. `list`, `deals`, `tasks`). |

### Return Value

`Promise<Result<T[]>>`{lang="ts-type"} ‚Äî a promise that resolves to an [`Result`{lang="ts-type"}](/docs/core/result/) object.

This object provides:

* `.getData(): Payload<T>`{lang="ts-type"} ‚Äî returns the full API response.
* `.isSuccess: boolean`{lang="ts-type"} ‚Äî flag indicating successful request execution.
* `.getErrorMessages(): string[]`{lang="ts-type"} ‚Äî array of error messages.

## Key Concepts

**Retrieving all data without pagination:** Set `start: -1` for efficient export of large data volumes. This disables the total count calculation, significantly speeding up the process.

* We will still receive 50 records per request.
* The idea is to change the filter condition on the ID field (`'>id': lastId`) and continue calling the request until an empty response is received.

### Error Handling

Always check the result using `isSuccess` and handle errors.

```ts
const response = await $b24.callFastListMethod('some.method', {}, 'id', 'items')

if (!response.isSuccess) {
  // Handling API error
  console.error(new Error(`API Error: ${response.getErrorMessages().join('; ')}`))
  return
}

// Working with a successful result
const data = response.getData()
```

## Alternatives and Recommendations

* **For working with very large lists:**: To save memory, use [`fetchListMethod`](/docs/hook/methods/fetch-list-method/) ‚Äî the method returns an asynchronous generator for stepping through large lists.
* **For batch operations:** Use [`callBatch`](/docs/hook/methods/call-batch/) to execute up to 50 commands in a single request.
* **On the client (browser):** Use the built-in [`B24Frame`](/docs/frame/) object instead of `B24Hook`.
