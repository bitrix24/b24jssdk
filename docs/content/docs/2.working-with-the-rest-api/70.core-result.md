`Result<T>`{lang="ts-type"} is the SDK's uniform return type for higher-level helpers (`callBatch`, `B24HelperManager.loadData`, `OptionsManager.save`, ŌĆ”). The class implements `IResult<T>`{lang="ts-type"}; the typed REST helpers (`call.make`, `callList.make`, ŌĆ”) return its specialised subclass [`AjaxResult<T>`](/docs/working-with-the-rest-api/core-ajax-result/).

## Usage

```ts
import { Result } from '@bitrix24/b24jssdk'

function parseConfig(raw: string): Result<{ theme: string }> {
  try {
    const data = JSON.parse(raw)
    if (typeof data?.theme !== 'string') {
      return Result.fail<{ theme: string }>('theme is required', 'config:theme')
    }
    return Result.ok({ theme: data.theme })
  } catch (error) {
    return Result.fail<{ theme: string }>(error as Error, 'config:json')
  }
}

const result = parseConfig(rawJson)
if (!result.isSuccess) {
  console.error(result.getErrorMessages())
  return
}
const { theme } = result.getData()!
```

## Constructor

```ts-type
new Result<T>(data?: T)
```

Default `data` is `null`{lang="ts-type"}.

## Static Factories

```ts-type
Result.ok<U>(data?: U): Result<U>
Result.fail<U>(error: Error | string, key?: string): Result<U>
```

## Getters

| Getter | Type | Description |
|----|----|----|
| `isSuccess` | `boolean`{lang="ts-type"} | `true` when the error map is empty. |
| `errors` | `Map<string, Error>`{lang="ts-type"} | Underlying error storage (read-only by convention). |

## Data Methods

```ts-type
setData(data: T | null | undefined): Result<T>
getData(): T | null | undefined
```

`setData` is chainable. Note: `AjaxResult`{lang="ts-type"} **disables** `setData` and throws `ReferenceError`{lang="ts-type"} on call.

## Error Methods

```ts-type
addError(error: Error | string, key?: string): Result<T>
addErrors(errors: (Error | string)[]): Result<T>
hasError(key: string): boolean
getErrors(): IterableIterator<Error>
getErrorMessages(): string[]
```

When `key` is omitted, `addError` generates a UUID v7 via [`Text.getUuidRfc4122()`](/docs/working-with-the-rest-api/tools-text/) so duplicate strings still get distinct slots.

## `toString`

```ts-type
toString(): string
```

Pretty-prints the result for logs:

- success ŌåÆ `Result(success): {<data JSON>}`
- failure ŌåÆ `Result(failure): {<data JSON>}\nErrors: <messages joined by ", ">`

`Error` instances inside the data are serialised as `{ name, message, stack }`. Failure to stringify falls back to `'[Unable to serialize data]'`.
