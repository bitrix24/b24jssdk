`useFormatter()`{lang="ts-type"} returns two shared formatters:

- `formatterNumber` — `FormatterNumbers` singleton, locale-aware number formatting.
- `formatterIban` — `FormatterIban` singleton, pre-loaded with **all** IBAN country specifications (and the regional French subdivisions: GF, GP, MQ, RE, PF, TF, YT, NC, BL, MF, PM, WF).

```ts
import { useFormatter } from '@bitrix24/b24jssdk'

const { formatterNumber, formatterIban } = useFormatter()

console.log(formatterIban.printFormat('GB29NWBK60161331926819'))
// 'GB29 NWBK 6016 1331 9268 19'

console.log(formatterIban.isValid('DE89370400440532013000'))
// true
```

::note
`useFormatter()` registers the IBAN specifications on every call, but the underlying `FormatterIban` is a singleton — adding a specification for the same country code is a no-op. Calling `useFormatter()` multiple times is cheap.
::

## `FormatterNumbers`

Locale-aware wrapper around `Intl.NumberFormat` (currency, decimal, percent). The instance is shared across the package; if you replace its locale, every consumer sees the change.

## `FormatterIban`

Validates and pretty-prints IBANs across the supported countries. Key methods:

- `isValid(iban: string): boolean` — checksum + length check.
- `printFormat(iban: string, sep?: string): string` — groups of four characters.
- `electronicFormat(iban: string): string` — strips whitespace, uppercases.
- `addSpecification(spec: IbanSpecification): void` — register a non-standard country.
