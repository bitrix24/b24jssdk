## What it does

Touches every read/write operation Bitrix24 Disk supports without multipart: `disk.storage.getlist`, `disk.folder.getchildren` (paged via `callListMethod`), `disk.folder.addsubfolder`, `disk.file.get`. Includes a `callBatch` example combining storages and root-folder listing in one round-trip.

## Stack

Node.js 18+. No external dependencies.

## Environment

```bash
export B24_HOOK='https://your.bitrix24.com/rest/1/secret'
```

## Run

```bash
npx tsx 05-disk-files.ts
```

## Notes

- Uploading a binary requires `disk.folder.uploadfile` with `multipart/form-data`. The SDK can do this via `Http.request`, but it isn't a drop-in `callMethod` call — that recipe is on the suggestion list.
- Disk methods use **classic** uppercase fields: `ID`, `NAME`, `PARENT_ID`, `ROOT_OBJECT_ID`. There is no v3 Disk API.
- `disk.folder.getchildren` is paged — always go through `callListMethod` (or `fetchListMethod` for large folders), not a single `callMethod`.
