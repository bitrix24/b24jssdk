`Text`{lang="ts-type"} is a singleton (`new TextManager()`) exported from `@bitrix24/b24jssdk`. It uses [Luxon](https://moment.github.io/luxon/) for date operations.

```ts
import { Text } from '@bitrix24/b24jssdk'

console.log(Text.getUuidRfc4122())          // '019323ac-8ace-725b-a3dc-6a7c333da066'
console.log(Text.getDateForLog())           // '2026-05-04 09:53:51'
console.log(Text.numberFormat(1234.567, 2)) // '1,234.57'
```

## Identifiers

| Method | Description |
|----|----|
| `getRandom(length = 8)` | Random `[a-z0-9]` string of given length. |
| `getUniqId()` | UUID v4 (best-effort, generated locally). |
| `getUuidRfc4122()` | UUID v7 (time-ordered). Used by the SDK as the default request id. |

## Entity Encoding

| Method | Description |
|----|----|
| `encode(value)` | Replaces `&`, `<`, `>`, `'`, `"` with their entity codes (without trailing `;`). |
| `decode(value)` | Inverse of `encode` — also handles numeric entities (`&#38`, `&#60`, …). |

::note
`encode` / `decode` deliberately omit the trailing `;` (e.g. `&amp` instead of `&amp;`). This matches the legacy Bitrix Framework behaviour.
::

## Type Conversion

```ts-type
toNumber(value: any): number          // 0.0 on parse failure
toInteger(value: any): number         // toNumber(parseInt(value, 10))
toBoolean(value: any, trueValues?: string[]): boolean // 'true'/'y'/'1'/1/true → true
```

## Case Helpers

```ts-type
toCamelCase(str: string): string
toPascalCase(str: string): string
toKebabCase(str: string): string
capitalize(str: string): string
```

`toCamelCase` recognises `-`, `_`, and whitespace as separators. `toPascalCase` is `capitalize(toCamelCase(str))`. `toKebabCase` splits on uppercase-letter boundaries (`getUserId` → `get-user-id`).

## Number Formatting

```ts-type
numberFormat(
  value: number,
  decimals?: number,        // default 0
  decPoint?: string,        // default '.'
  thousandsSep?: string     // default ','
): string
```

Same algorithm Bitrix24 uses on the server — non-finite values become `0`, fraction is rounded, the thousands separator inserts every three digits left of the decimal point.

## Date Helpers (Luxon)

```ts-type
toDateTime(dateString: string, template?: string, opts?: DateTimeOptions): DateTime
toB24Format(date: string | Date | DateTime): string
getDateForLog(): string  // 'yyyy-MM-dd HH:mm:ss'
```

`toDateTime` uses `DateTime.fromFormat` when `template` is provided, otherwise `DateTime.fromISO`. `toB24Format` outputs the format Bitrix24 expects in REST payloads: `yyyy-MM-dd'T'HH:mm:ssZZ`.

## Query String

```ts-type
buildQueryString(params: any): string
```

Builds an `application/x-www-form-urlencoded` string from a plain object. Array values become `key[i]=value` pairs.
