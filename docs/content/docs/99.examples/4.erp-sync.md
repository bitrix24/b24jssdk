## What it does

Pulls contacts from both Bitrix24 (via `fetchListMethod`) and an external ERP (mocked in the recipe — replace with your client). Matches them by tax id (`ufCrmInn`) primarily and email as a fallback. Creates missing rows on each side and syncs changed name fields back to Bitrix24.

## Stack

Node.js 18+, `node-cron`.

## Install

```bash
pnpm add node-cron
```

## Environment

```bash
export B24_HOOK='https://your.bitrix24.com/rest/1/secret'
```

## Run

```bash
npx tsx 04-erp-sync.ts
```

## Source

[`.claude/skills/b24jssdk-recipes/examples/04-erp-sync.ts`](https://github.com/bitrix24/b24jssdk/blob/main/.claude/skills/b24jssdk-recipes/examples/04-erp-sync.ts).

## Notes

- ERP-side functions (`fetchErpContacts`, `createErpContact`, `updateErpContact`) are stubs. Plug in your client (1C OData, REST, etc.).
- The matching key is `ufCrmInn` — adjust the field name to whatever your custom-field namespace uses.
- For thousands of contacts use `callBatchByChunk` for the bulk-create step instead of awaiting per row.
